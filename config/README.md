# config

Centralized configuration files consumed by all pipeline modules. JSON format throughout.

## Files

### asset_universe.json

Flat list of 67 asset pairs (e.g., `"BTC/USD"`, `"FET/USD"`) that define the tradable universe. All other config files reference assets from this list.

### sector_map.json

Maps each of the 67 tokens to one of 6 canonical sectors, with optional dual-routing to a secondary sector.

```json
{
  "BTC/USD": {"primary": "l1_infra", "secondary": "store_of_value"},
  "NEAR/USD": {"primary": "l1_infra", "secondary": "ai_compute"},
  "FET/USD": {"primary": "ai_compute", "secondary": null}
}
```

**Sectors:** `l1_infra`, `defi`, `ai_compute`, `meme`, `store_of_value`, `other`

**Dual-routing:** Two tokens have secondary sectors: BTC (l1_infra + store_of_value) and NEAR (l1_infra + ai_compute). The composite adapter applies secondary sector signals at 50% weight.

Generated by `scripts/build_sector_map.py` from CoinGecko category data.

### sector_config.json

Per-sector parameters controlling sentiment aggregation behavior:

| Parameter | Description | Example (meme) | Example (l1_infra) |
|-----------|-------------|-----------------|---------------------|
| `sentiment_weight` | Weight of sentiment in composite score | 0.9 | 0.2 |
| `lookback_hours` | Time window for article aggregation | 4 | 24 |
| `decay_lambda` | Exponential decay rate (higher = faster decay) | 2.0 | 0.1 |
| `momentum_lookback_hours` | Window for momentum computation | 4 | 24 |
| `entry_threshold` | Minimum signal strength to act on | 0.2 | 0.3 |
| `catalyst_threshold` | Minimum sentiment for catalyst detection | 0.5 | 0.7 |
| `regime` | Behavioral classification | pure_sentiment | persistent_co_mover |

**Regime types:** `persistent_co_mover`, `event_driven`, `pure_sentiment`, `weakly_connected`, `mixed`

### sources.json

Registry of news data sources with fetch configuration:

| Source ID | Type | Priority | Source Weight | Config |
|-----------|------|----------|---------------|--------|
| `cryptopanic` | api | 1 | 1.0 | API base URL, auth token env var |
| `rss_coindesk` | rss | 2 | 0.8 | Feed URL |
| `rss_cointelegraph` | rss | 2 | 0.8 | Feed URL |
| `rss_decrypt` | rss | 2 | 0.8 | Feed URL |

Each source has `enabled` (boolean), `priority` (int), `source_weight` (float), and `fetch_interval_min` (int) fields.

### composite_config.json

Controls the composite scoring system:

**Signal weights** (must sum to 1.0):
- `technical`: 0.35
- `derivatives`: 0.25
- `on_chain`: 0.15
- `multi_timeframe`: 0.10
- `sentiment`: 0.15

**Decision thresholds:**
- `buy_default`: 0.3 (composite score > 0.3 = BUY)
- `sell_default`: 0.3 (composite score < -0.3 = SELL)

**Override parameters:** Funding rate thresholds (soft/hard), NUPL thresholds (soft/hard), penalty multipliers, catalyst boost multiplier, timeframe opposition multiplier.

### derivatives_config.json

Configuration for the derivatives scoring module:

- `polling_interval_seconds`: 300 (5 minutes)
- `funding_rate_neutral_band`: [-0.0001, 0.0001]
- `oi_lookback_periods`: 12
- `long_short_extreme_threshold`: 2.0
- **Sub-weights:** funding=0.4, oi_divergence=0.35, long_short=0.25

### onchain_config.json

Configuration for the on-chain analytics module:

- `coinmetrics_poll_hour_utc`: 0 (midnight UTC)
- `whale_transfer_threshold_usd`: 1,000,000
- `whale_rolling_window_hours`: 4
- `exchange_flow_normalization_days`: 30
- `nupl_thresholds`: euphoria=0.75, capitulation=0.0
- **Sub-weights:** exchange_flow=0.3, nupl=0.25, active_addresses=0.15, whale_activity=0.3

## How to Modify

1. **Add a new token:** Add the pair to `asset_universe.json`, then run `scripts/build_sector_map.py` to auto-classify it, or manually add an entry to `sector_map.json`.
2. **Add a new sector:** Add the sector to `sector_config.json` with appropriate parameters. Update `sector_map.json` to assign tokens. Update `VALID_SECTORS` in `news_sentiment/prompter.py`.
3. **Add a news source:** Add an entry to the `sources` array in `sources.json` with `type` set to `"api"` or `"rss"`. Implement a fetcher in `news_sentiment/processors.py` if the type is new.
4. **Tune scoring weights:** Edit `composite_config.json` weights (ensure they sum to 1.0) or override thresholds.
